COMPORTAMENTO CORRETO ESPERADO:

Sessão deve permanecer ativa até finalização manual

"Continuar sessão" deve sempre recuperar sessão não finalizada

1. VERIFICAR LÓGICA DE DETECÇÃO DE SESSÃO ATIVA:

2. VERIFICAR CRITÉRIO DE SESSÃO ATIVA:

Sessão ativa = endTime nulo ou undefined
NÃO deve considerar data como critério para encerrar sessão
Verificar se não há lógica que encerra sessões automaticamente

3. VERIFICAR BOTÃO "CONTINUAR NOVA SESSÃO":

Deve procurar PRIMEIRO por sessão ativa existente
Se encontrar: redirecionar para sessão ativa

Console.log da lógica de decisão

4. VERIFICAR PERSISTÊNCIA DA SESSÃO:

Confirmar se sessão está sendo salva corretamente no banco
Verificar se não há cleanup automático de sessões
Ver se sessão permanece no banco após mudança de data

5. IMPLEMENTAR RECUPERAÇÃO ROBUSTA:
Lógica correta:
1. Buscar sessão com endTime = null para o usuário
2. SE encontrou: redirecionar para sessão ativa
3. SE não encontrou: permitir criar nova sessão
4. NUNCA encerrar sessão automaticamente
6. VERIFICAR TIMEZONE/DATA:

Ver se mudança de fuso horário afeta busca
Confirmar se comparação de datas não interfere
Verificar se sessão não é marcada como "expirada"

7. ADICIONAR PROTEÇÕES:

Aviso antes de criar nova sessão se já existe uma ativa
Opção de recuperar sessão anterior
Confirmação antes de abandonar sessão não finalizada
