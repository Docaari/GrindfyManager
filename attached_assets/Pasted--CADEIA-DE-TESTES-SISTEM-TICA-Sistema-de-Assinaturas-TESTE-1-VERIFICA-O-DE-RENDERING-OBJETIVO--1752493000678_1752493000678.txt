🧪 CADEIA DE TESTES SISTEMÁTICA: Sistema de Assinaturas
TESTE 1: VERIFICAÇÃO DE RENDERING
OBJETIVO: Identificar loops infinitos de re-render
EXECUTAR:

Abrir console do navegador (F12)
Navegar para aba "Assinaturas" na gestão de usuários
Verificar se há erros ou warnings no console
Observar se componentes estão re-renderizando constantemente
Console.log dos estados que estão mudando

IDENTIFICAR:

Quais componentes estão causando o loop
Que estado está sendo atualizado infinitamente
Se há useEffect sem dependências corretas


🧪 TESTE 2: VERIFICAÇÃO DE QUERIES
OBJETIVO: Verificar se dados estão sendo buscados corretamente
EXECUTAR:

Verificar se queries de assinatura estão funcionando
Console.log dos dados retornados pela API
Verificar se dados chegam no formato esperado
Testar se refetch está sendo chamado adequadamente

IDENTIFICAR:

Se API retorna dados corretos
Se estrutura de dados está adequada
Se há conflito entre diferentes queries


🧪 TESTE 3: VERIFICAÇÃO DE ESTADO
OBJETIVO: Identificar problemas de gerenciamento de estado
EXECUTAR:

Console.log de todos os estados do componente
Verificar se useState está sendo usado corretamente

Verificar se há atualizações de estado condicionais

IDENTIFICAR:

Quais estados estão mudando constantemente
Se há dependencies missing em useEffect
Se há setState em loops


🧪 TESTE 4: VERIFICAÇÃO DE PROPS
OBJETIVO: Verificar se componentes recebem props estáveis
EXECUTAR:

Console.log de todas as props passadas
Verificar se props estão mudando a cada render

IDENTIFICAR:

Props que mudam desnecessariamente

🧪 TESTE 5: VERIFICAÇÃO DE EFFECTS
OBJETIVO: Identificar useEffect problemáticos
EXECUTAR:

IDENTIFICAR:

useEffect com dependências incorretas
Efeitos que causam cascata de atualizações
Missing dependencies
