🔒 REGRAS CRÍTICAS PARA MVP:

BACKUP obrigatório antes de qualquer mudança

🔍 Problema Identificado
📊 Situação:

Logs mostram: foreign key constraint errors constantes
Tabela afetada: access_logs
Impacto: Integridade de dados comprometida
Funcionamento: Sistema funciona normalmente (erros apenas nos logs)

🎯 Objetivo da Correção:

Eliminar errors nos logs
Garantir integridade referencial
Manter todas funcionalidades de auditoria

🔧 Abordagem de Correção
Passo 1: Investigação Segura

Identificar exatamente quais FKs estão com problema
Verificar estrutura atual da tabela access_logs
Mapear relacionamentos incorretos
Documentar antes de fazer mudanças

Passo 2: Backup Obrigatório

Backup completo do banco antes de iniciar
Export da tabela access_logs especificamente
Verificar backup está íntegro
Confirmar possibilidade de rollback

Passo 3: Correção Incremental

Corrigir uma FK por vez
Testar após cada correção
Verificar que logs param de mostrar erros
Confirmar que audit trails funcionam

Passo 4: Validação Completa

Teste de login/logout funcionando
Verificar logs sendo gravados corretamente
Confirmar que não há novos erros
Audit trail preservado

🧪 Protocolo de Teste Rigoroso
Após cada mudança na FK:

 Sistema de login funciona normalmente
 Logout não apresenta erros
 Access logs são gravados sem erro
 Audit trail está completo
 Console logs não mostram FK errors
 Performance não foi afetada

Se algum teste falhar:

PARAR imediatamente
ROLLBACK para backup
Reportar problema específico
Aguardar nova estratégia
