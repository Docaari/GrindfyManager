🧠 Prompt para Replit Agent – Interpretação de TXT da Rede Coin
Você deve implementar a importação de um arquivo .txt da rede Coin, seguindo estas instruções com precisão. Este arquivo não é tabular, mas sim composto por linhas de texto soltas. Cada linha representa uma ação de torneio:

📄 Formato do Arquivo
Cada linha representa uma transação da conta (entrada ou saída).

As linhas que contêm Withdrawal representam Buy-ins (ou reentradas).

As linhas que contêm Deposit representam a conclusão do torneio com a premiação recebida (mesmo que seja zero).

O par Withdrawal → Deposit indica um único torneio: a primeira linha representa o buy-in e a segunda o resultado.

✅ Dados a serem extraídos por torneio:
Campo	Regras específicas
Nome do Torneio	Está logo após o símbolo ?, até o início da data. Pode conter espaços e hífens.
Data do Torneio	Extraída da linha de Withdrawal. Sempre no formato YYYY-MM-DD e deve ser convertido para DD/MM/YYYY.
Buy-in	Valor da linha de Withdrawal, em USDT (ex: Withdrawal-25 USDT)
Prêmio	Valor da linha de Deposit, em USDT (ex: Deposit 131.25 USDT)
Posição	Sempre N/A, pois o arquivo não contém essa informação
Rede/Site	Coin, pois esse formato é exclusivo dessa rede
Categoria	Se a linha contiver "PKO", classificar como PKO. Caso contrário, classificar como Vanilla
Velocidade	Use palavras-chave no nome para definir:
Hyper → se contiver Hyper
Turbo → se contiver Sprint, Turbo, Bolt, Rápido ou Flash
Normal → se não contiver nenhuma dessas

🕓 Formato da Data
Presente no final da linha (ex: 2025-01-02 22:10:38).

Utilize apenas a parte da data (YYYY-MM-DD), convertendo para o formato DD/MM/YYYY.

🔍 Delimitadores
O arquivo não utiliza delimitadores tabulares. A extração deve ser feita com expressões regulares (regex) e divisão de strings.

⚠️ Tratamento de Erros
Caso uma linha não contenha os dados esperados, ignore a linha, e continue a importação.

Se houver Deposit sem Withdrawal correspondente anterior, ignorar.

Se houver Withdrawal sem Deposit posterior, também ignorar.

📌 Considerações adicionais
Rake: Pode ser definido como 0.00 (não consta no arquivo).

Reentradas: Ignorar, não há marcação clara para reentradas múltiplas — cada entrada será tratada como torneio separado.

O sistema deve parear corretamente cada Withdrawal com seu respectivo Deposit subsequente (ordem importa).
